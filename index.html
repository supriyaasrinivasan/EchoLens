HTML Introduction

Learn the fundamental building blocks of web development

New to HTML?

Don't worry! HTML is like building with blocks. Here's what you need to know:

  • HTML = HyperText Markup Language
  • It's used to create web pages
  • Tags look like <this>
  • Most tags have opening and closing parts

Quick Start Guide

Here's your first HTML page:

<!DOCTYPE html> <html> <head> <title>My First Page</title> </head> <body> <h1>Hello World!</h1> <p>This is my first webpage.</p> </body> </html>

What are HTML Tags?

HTML tags define the structure and content of a web page. They are enclosed in angle brackets and tell the browser how to display content.

<tagname> Content </tagname>

Tag Syntax

Tags typically come in pairs with opening and closing tags:

  • <tagname> — Opening Tag
  • </tagname> — Closing Tag

Basic HTML Structure

Every HTML document follows this basic structure:

<!DOCTYPE html> <html> <head> <title>Page Title</title> </head> <body> <h1>Hello World!</h1> </body> </html>

Essential Tags

Core HTML elements every developer should know:

  • <!DOCTYPE html> – Declares HTML5
  • <html> – Root element
  • <head> – Contains metadata
  • <title> – Sets browser title
  • <body> – Page content

Content Tags

Tags for structuring your page content:

  • <h1> to <h6> – Headings
  • <p> – Paragraph
  • <a> – Anchor (link)
  • <img> – Image
  • <br> – Line break

List Tags

Organize content with lists:

  • <ul> – Unordered List
  • <ol> – Ordered List
  • <li> – List Item

Text Formatting Tags

Style and format your text content:

  • <strong> – Bold (important)
  • <em> – Italic (emphasis)
  • <mark> – Highlighted text
  • <small> – Small text
  • <del> – Deleted text
  • <ins> – Inserted text

Semantic HTML5 Tags

Modern semantic elements for better structure:

  • <header> – Page/section header
  • <nav> – Navigation links
  • <main> – Main content area
  • <section> – Content section
  • <article> – Independent content
  • <aside> – Sidebar content
  • <footer> – Page/section footer

HTML Comments

Add notes to your code that won't display on the page:

<!-- This is a comment -->

Use cases: Code documentation, temporary content removal, notes for developers

HTML Head & Metadata

Essential metadata and document configuration

The <head> Tag

The head tag contains metadata and resources that are not displayed directly on the page but help browsers, search engines, and social media understand your content.

<head> <!-- Metadata goes here --> </head>

Meta Tags

Essential meta tags for modern web development:

<meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <meta name="description" content="Page description"> <meta name="keywords" content="html, tutorial, guide"> <meta name="author" content="Your Name">

External Resources

Link external CSS files and set page icons:

<link rel="stylesheet" href="styles.css"> <link rel="icon" href="favicon.ico" type="image/x-icon">

Internal Styles & Scripts

Embed CSS and JavaScript directly in the head:

<style> body { background-color: #f2f2f2; } </style> <script> console.log("Hello from the head!"); </script>

Social Media Meta Tags

Open Graph and Twitter Card meta tags for social sharing:

<!-- Open Graph --> <meta property="og:title" content="Page Title"> <meta property="og:description" content="Page description"> <meta property="og:image" content="image.jpg"> <meta property="og:url" content="https://example.com"> <!-- Twitter Cards --> <meta name="twitter:card" content="summary_large_image"> <meta name="twitter:title" content="Page Title"> <meta name="twitter:description" content="Description">

Performance & SEO Meta Tags

Tags for better performance and search engine optimization:

<!-- Preload critical resources --> <link rel="preload" href="font.woff2" as="font" type="font/woff2" crossorigin> <!-- DNS prefetch for external domains --> <link rel="dns-prefetch" href="//fonts.googleapis.com"> <!-- Canonical URL --> <link rel="canonical" href="https://example.com/page"> <!-- Robots meta --> <meta name="robots" content="index, follow">

HTML Attributes

Add behavior and functionality to your HTML elements

What are Attributes?

HTML Attributes provide additional information about HTML elements to define their behavior, appearance, or functionality.

<tagname attribute="value">Content</tagname>

Global Attributes

Attributes that can be used with any HTML element:

Attribute Description Example
id Unique identifier <div id="header">
class CSS class name <p class="note">
style Inline CSS styling <h1 style="color:blue;">
title Tooltip text on hover <a title="Go home">

Image Attributes

Essential attributes for the <img> tag:

  • src: Image file path
  • alt: Alternative text for accessibility
  • width, height: Image dimensions
  • title: Tooltip text
<img src="logo.jpg" alt="Company Logo" width="200" height="100">

Form Input Attributes

Attributes that control form behavior:

  • type: Input type (text, password, email)
  • name: Form submission identifier
  • value: Default value
  • placeholder: Hint text
  • required: Makes field mandatory
  • readonly: Prevents editing
  • disabled: Disables the field
  • maxlength: Character limit

Anchor Attributes

Essential attributes for anchor tags:

Attribute Description Example
href Link destination href="page.html"
target Where to open link target="_blank"
rel Relationship between pages rel="noopener"
download Force file download download="filename.pdf"

Accessibility Attributes

ARIA attributes for better accessibility:

  • aria-label: Accessible name for element
  • aria-describedby: References description
  • role: Element's purpose
  • tabindex: Tab order control
  • aria-hidden: Hide from screen readers
  • aria-expanded: Collapsible state
<button aria-label="Close dialog" aria-expanded="false"> <img src="icon.png" alt="Close" aria-hidden="true"> </button>

Block vs Inline Elements

Understanding how HTML elements behave and display

Block-Level Elements

Elements that create structure and layout:

  • Always start on a new line
  • Take up 100% of the width
  • Can contain inline or other block elements
  • Used for page structure

Examples: <div>, <p>, <h1-h6>, <section>, <article>

Inline Elements

Elements for styling text within blocks:

  • Do NOT start on a new line
  • Only take up necessary width
  • Cannot contain block-level elements
  • Used for text styling

Examples: <span>, <a>, <strong>, <em>, <img>

Comparison Table

Feature Block Element Inline Element
New line ✅ Yes ❌ No
Full width ✅ Yes ❌ Content width only
Can contain blocks ✅ Yes ❌ No
Primary use Structure/Layout Text styling

Tables and Lists

Organize and structure your data effectively

Unordered Lists (<ul>)

Create bulleted lists when order doesn't matter:

<ul> <li>Item 1</li> <li>Item 2</li> <li>Item 3</li> </ul>

Use cases: Navigation menus, feature lists, grocery lists

Ordered Lists (<ol>)

Create numbered lists when sequence matters:

<ol> <li>First step</li> <li>Second step</li> <li>Third step</li> </ol>

Attributes:

  • type: Number style (1, A, a, I, i)
  • start: Starting value
  • reversed: Count down

Description Lists (<dl>)

Create term and definition pairs:

<dl> <dt>HTML</dt> <dd>HyperText Markup Language</dd> <dt>CSS</dt> <dd>Cascading Style Sheets</dd> </dl>

Use cases: Glossaries, FAQs, metadata

Tables (<table>)

Organize data in rows and columns:

<table> <thead> <tr> <th>Name</th> <th>Age</th> </tr> </thead> <tbody> <tr> <td>John</td> <td>25</td> </tr> </tbody> </table>

Security & Performance Best Practices

Important considerations when embedding content:

  • Always validate sources: Only embed content from trusted domains
  • Use sandbox attributes: Restrict iframe capabilities
  • Implement CSP: Content Security Policy headers
  • Loading optimization: Use lazy loading for performance

What is JavaScript?

JavaScript is a programming language that enables interactive web content like:

  • Dynamic HTML updates
  • User interaction handling
  • Form validation
  • Animations and effects
  • Data manipulation

Development Environment

You can write and run JavaScript code using:

  • Browser Console: Press F12 or right-click and select "Inspect" to open DevTools
  • Text Editor/IDE: Use Visual Studio Code or similar for writing scripts
  • Online Playgrounds: Try CodePen or JSFiddle for quick experiments

  • let allows declaring variables that can be changed later

If Statements

Execute code based on specific conditions using if, else if, and else statements.

let age = 18; // Basic if statement if (age >= 18) { console.log("You are an adult"); } else { console.log("You are a minor"); } // if-else if-else let score = 85; if (score >= 90) { console.log("Grade: A"); } else if (score >= 80) { console.log("Grade: B"); } else { console.log("Grade: C or below"); }
  • Use if for a single condition
  • Add else if for additional conditions
  • Use else for all other cases

Loops

Loops execute the same block of code repeatedly based on conditions:

  • for loop: Used when the number of iterations is known
  • while loop: Repeats while a condition is true
  • do-while loop: Executes once before checking the condition
// For loop to print numbers 1 to 5 for (let i = 1; i <= 5; i++) { console.log(i); } // While loop let n = 1; while (n <= 5) { console.log(n); n++; } // Do-while loop let m = 1; do { console.log(m); m++; } while (m <= 5);

Arrays

Arrays store multiple values in a single variable:

  • Ordered collection of values
  • Zero-based indexing
  • Can store mixed data types
  • Dynamic length
// Create an array let fruits = ["apple", "banana", "cherry"]; console.log(fruits[0]); // apple console.log(fruits.length); // 3 // Add element fruits.push("date"); console.log(fruits); // Loop through array for (let i = 0; i < fruits.length; i++) { console.log(fruits[i]); }

Objects

Objects store data as key-value pairs:

  • Properties defined as key-value pairs
  • Access values using dot notation or brackets
  • Great for representing real-world entities
  • Keys must be unique within an object
// Create an object let person = { name: "Alice", age: 30, city: "New York" }; console.log(person.name); // Alice console.log(person["age"]); // 30

Type Conversion and Coercion

JavaScript can convert values from one type to another:

  • Type Coercion: Automatic type conversion
  • Type Conversion: Explicit conversion using methods
  • Different operators can cause different coercion rules
// Type coercion (automatic) console.log("5" + 2); // "52" (string concatenation) console.log("5" - 2); // 3 (string converted to number) // Explicit type conversion console.log(Number("5")); // 5 console.log(String(5)); // "5" console.log(Boolean(0)); // false (explicit conversion) console.log(Boolean(1)); // true console.log(Boolean("")); // false console.log(Boolean("hi")); // true

Functions and Scope

Functions are reusable blocks of code designed to perform a specific task

Function Basics

Functions are building blocks of reusable code:

// Function Declaration function greet(name) { return "Hello, " + name + "!"); } // Function Expression const add = function(a, b) { return a + b; }; // Arrow Function const multiply = (x, y) => x * y; // Calling Functions console.log(greet("John")); // Hello, John! console.log(add(5, 3)); // 8 console.log(multiply(4, 2)); // 8

Parameters and Arguments

Functions can take inputs (parameters) and return outputs:

// Parameters with default values function welcome(name = "Guest") { return `Welcome, ${name}!`; } // Multiple parameters function calculateArea(length, width) { return length * width; } // Rest parameters function sum(...numbers) { return numbers.reduce((total, num) => total + num, 0); } console.log(welcome()); // Welcome, Guest! console.log(calculateArea(5, 3)); // 15 console.log(sum(1, 2, 3, 4)); // 10

Objects, Methods, and Prototypes

Objects can have methods and inherit properties through prototypes:

// Object with methods const person = { name: "Alice", age: 25, greet: function() { console.log("Hello, " + this.name); } }; person.greet(); // Output: Hello, Alice // Constructor function function Animal(name) { this.name = name; } // Adding method to prototype Animal.prototype.speak = function() { console.log(this.name + " makes a sound"); }; // Inheritance function Dog(name, breed) { Animal.call(this, name); // Call parent constructor this.breed = breed; } // Set up prototype chain Dog.prototype = Object.create(Animal.prototype); Dog.prototype.constructor = Dog; // Add method specific to Dog Dog.prototype.bark = function() { console.log("Woof!"); }; const rex = new Dog("Rex", "German Shepherd"); rex.speak(); // Rex makes a sound rex.bark(); // Woof!
  • Objects can have methods (functions as properties)
  • Prototypes enable inheritance between objects
  • Constructor functions create object templates
  • The this keyword refers to the current object

Asynchronous JavaScript

Handle operations that take time without blocking code execution:

// Promise example function fetchData() { return new Promise(resolve => { setTimeout(() => resolve("Data received"), 1000); }); } fetchData().then(data => console.log(data)); // Using async/await async function getData() { const result = await fetchData(); console.log(result); } getData(); // Error handling with try/catch async function fetchWithError() { try { const data = await fetchData(); console.log(data); } catch (error) { console.error("Error:", error); } }
  • Callbacks: Functions passed as arguments
  • Promises: Handle future values
  • async/await: Write cleaner async code
  • Error handling with try/catch

Modules and Import/Export

Organize code into separate files:

// math.js - Exporting export function add(a, b) { return a + b; } export const PI = 3.14159; export default class Calculator { multiply(a, b) { return a * b; } } // main.js - Importing import Calculator, { add, PI } from './math.js'; console.log(add(2, 3)); // 5 console.log(PI); // 3.14159 const calc = new Calculator(); console.log(calc.multiply(4, 2)); // 8

  • Use import to use exported code
  • One default export per module
  • Multiple named exports allowed

Browser Storage APIs

Store data locally in the user's browser:

// localStorage - persists even after browser is closed // Store data localStorage.setItem("username", "Alice"); localStorage.setItem("isLoggedIn", "true"); // Retrieve data const username = localStorage.getItem("username"); console.log(username); // Alice // Remove single item localStorage.removeItem("username"); // Clear all stored data localStorage.clear(); // sessionStorage - clears when browser session ends sessionStorage.setItem("tempData", "Session value"); console.log(sessionStorage.getItem("tempData")); // Store complex data (objects/arrays) const user = { name: "Bob", age: 25, roles: ["admin", "editor"] }; localStorage.setItem("user", JSON.stringify(user)); // Retrieve and parse complex data const storedUser = JSON.parse(localStorage.getItem("user")); console.log(storedUser.name); // Bob console.log(storedUser.roles[0]); // admin
  • localStorage: Persists even after browser is closed
  • sessionStorage: Clears when browser session ends
  • Only stores strings (use JSON for complex data)
  • Storage limit is typically around 5-10MB
  • Useful for user preferences, form data, authentication tokens
